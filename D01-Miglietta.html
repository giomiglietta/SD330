Usability

  • Know the user you are designing for
  • Observe reactions to the system
  • Fix problems as they are found
  • Consider physical or mental limitations for the user
  • Be usable (effective, efficient, safe, good utility, easy to learn and remember)

Basic Design Principles

  • User should recieve feedback from input
  • Restrict the user interaction that can take place in certain moments
  • Interfaces should be consistent through system
  • Develop buttons and features that allow affordance

Interaction Design

  • Design alternatives
  • Develop prototypes for testing
  • Interfaces should be consistent through system
  • Evaluating the effectiveness of your product

Cognitive Design

  • Demonstrate to user how to interact with system (instructing, conversing, etc)
  • Ask the right questions to help design teams
  • Be aware of how the user will think about the system(experiential vs reflective)
  • Structure information in a way that makes it effective and easy to read

Design implications

  • Consider context
  • Avoid cluttering of visual interfaces
  • Use separators and white space to separate information
  • Design icons to make it easily readable
  • Use different sound effects to distinguish between different actions
  • Make the system easy to remember.
  • Use expressive interfaces to let the user know what is going on

Easy of use Design

  • Reduce cognitive load by making processes simple
  • Promote recognition
  • Allow users to easily label and organize to their liking
  • Create clear and easy to follow instructions
  • Have appropriate online help and tutorials
  • Let the user discover what options are available

Cultural Considerations

  • Some society read from right to left
  • Some societies like websites that are more cluttered compared to others
  • Some words, sayings or symbols may be offensive in certain cultures
